Information story

As a Seattle resident

I need a trustworthy feed of nearby independent businesses

So that I can discover and support local shops (not only chains)

Community goal: keep spending and jobs in the neighborhood

Existing information (before)

  • Yelp Fusion API
  • HTML chamber directories
  • Social / Instagram
  • I7 flat JSON file
  • No single portable schema
  • License / redistribution limits
  • Hard to query across sources
  • No quality or provenance block

What we improved (4 dimensions)

DimensionChange
Informationquality_flags, completeness score
StructureSBP envelope + LocalBusiness-aligned records
FormatVersioned JSON + published schema
AccessREST API (was static file / memory)

New structure: response envelope

{
  "schema_version": "1.0",
  "provenance": { "source", "license", "retrieved_at", ... },
  "query_summary": { "quality_summary": { "completeness_score" } },
  "businesses": [ { "id", "name", "categories", "quality_flags", ... } ]
}

API endpoints (demo)

GET /health — liveness

GET /schema — JSON Schema contract

GET /businesses/search?term=&location= — portable search

GET /businesses/{id} — single record

GET /stats — analyze by category

GET /open-now — hours-based filter

Replace base URL with your ngrok HTTPS URL during live demo

Live demo

Open in browser (start Flask + ngrok first):

/health

/businesses/search?term=coffee&location=Capitol Hill

/stats

Set base: ?api=https://YOUR.ngrok.app in this page URL to rewrite links

Security

  • contact email/phone = restricted by default
  • Authorization: Bearer <token> unlocks full contact
  • No API secrets in repo or responses
  • Documented in G9 test plan + pytest

Quality and performance

  • Average completeness_score ~0.92 (demo dataset, n=6)
  • 9 pytest tests + jsonschema validation
  • Warm search latency < 50 ms locally
  • Remediation: live Yelp adapter, CI uptime checks
