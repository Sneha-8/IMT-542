LocalFind

Helping Seattle residents discover and support independent neighborhood businesses through portable, trustworthy information.

Sneha Reddy · IMT 542 · Spring 2026

Meet Alex

Alex lives in Capitol Hill and wants to grab coffee from a local shop — somewhere with vegan options and Wi-Fi — not another chain. They open Yelp, scroll past sponsored results, jump to Instagram, then give up and default to Starbucks.

The information exists. It is just scattered, locked in apps, and hard to combine for a simple decision.

Why this matters

  • Money spent locally circulates longer in the neighborhood economy.
  • Small businesses lack the SEO and ad budgets of national chains.
  • Civic groups, newsletters, and accessibility tools need machine-readable listings — not screenshots of HTML directories.

We set out to turn fragmented business information into one portable product: LocalFind.

Where the information lives today

Sources people use

  • Yelp and Google Maps (proprietary, ad-driven)
  • Chamber of commerce HTML pages
  • Instagram and owner websites
  • Word of mouth

What goes wrong

  • No stable ID across systems
  • Cannot filter “open now + wifi + sustainable” in one query
  • No provenance — hard to trust for civic reuse
  • Owner contact mixed with public fields

The information journey we designed for

Curated & public sources → Normalize to one schema → Store in document database → REST API → LocalFind experience

Each step adds structure so the same facts can power a website, a map layer, a neighborhood newsletter, or an AI assistant — without re-scraping five sites.

What LocalFind does for people

Residents

Search by keyword, neighborhood, and tags. See ratings, hours, and whether a shop is open today. Support businesses that match their values.

Owners & partners

One consistent public listing. Sensitive contact details stay protected unless the viewer is authorized.

Behind the scenes, developers get versioned JSON, a published schema, and provenance on every response.

From scattered records to one portable shape

We did not just move a JSON file online. We redesigned how information is packaged:

  • Envelope — every response explains schema version, where data came from, and how complete it is.
  • Records — each business has split addresses, normalized phones, tags, hours, products, and quality indicators.
  • Access — HTTPS search and lookup endpoints instead of a static file only a developer can read.

Same facts, but structured so humans and software can rely on them.

Trust built into every response

  • Provenance — source, retrieval time, and license travel with the data.
  • Quality signals — completeness score and per-field flags so consumers know what is missing before they act.
  • Privacy — email and phone hidden from public API calls; unlocked only with an authorized token for owners or partners.

This is how we earn reuse by community apps without surprising business owners.

How we built it over the quarter

JSON file + Flask API → MongoDB + richer queries → Schema v1.0 + normalizer → LocalFind UI prototype

Early versions proved access patterns. The final system adds portability metadata, automated tests, and the experience you just saw.

Under the hood (for the curious)

  • Python Flask API with search, stats, and open-now endpoints
  • MongoDB (or file-based Mongita for easy local demos)
  • Normalizer transforms source JSON into Small Business Platform records
  • Nine automated tests validate schema and security behavior

Demo dataset: six real-shaped Seattle businesses; production path integrates Yelp Fusion with the same normalizer.
